Synopsis

The project is all about Text Mining using the dataset from blogs, twitter and news feeds and predict next word for giiven set of words.

  • It uses R package TM to perform various text mining operations such as pruning punctuations, stopwords and swearwords
  • At the heart of it is the ngram creation which is series of words combined in various meaningful permutation & combinations
  • It is this dictionary of words that is searched against for the given sentence to predict the next word or words in the higher order frequencies

About the App

  • Application (app) is built using Shinyapps and launced in shinyapps.io for public access
  • The app is backed by ngrams dictionary (with about 69 million records of quadgrams) built using the 3 datasets
  • Uses in-memory lightweight database SQLLite for querying the ngram dictionary

Application Algorithm & Insights

  • Ngram dictionary was built one-time in SQLLite in batch mode to avoid choking in CPU and Memory
  • To have a balance, we created quadgrams versus higher or lower numbered-grams
  • When set of words are entered in the text box and clicked “predict next word' button, the server uses SQLLite to query the ngrams dictionary for matching words, sort top 10 (configurable) matches by higher frequencies and finally, present list of next word(s), i.e. next to the last word of given sentence
